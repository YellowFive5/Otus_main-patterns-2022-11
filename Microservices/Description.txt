
Описание:
    Web/Desktop/Mobile clients - клиенты доступа для пользователей
    UserHost - api для клиентов пользователей
    StaffClient - клиент доступа для сотрудников / администраторов
    StaffHost - api для клиентов сотрудников / администраторов
    AuthorizationService - сервис авторизации и аутентификации
    InfoService - сервис доступа для сбора доступной для пользователя информации
    MessageService - сервис оповещения о событиях
    ResultsService - сервис результатов проведенных боев
    RatingService - сервис расчета рейтингов пользователей
    ReplayService - сервис записей проведенных боев
    OfficialTournamentService - сервис создания официальных боев и турниров
    CustomTournamentService - сервис создания пользовательских боев и турниров
    BattleExecutorService - сервис расчёта хода и результатов боя
    MessageBroker - общая шина сообщений для асинхронной связи сервисов 
 
Узкие места и потенциальные проблемы:
    
    1.  Масштабирование UserHosts по мере роста пользователей
    2.  Версионирование api UserHosts и клиентов пользователей
    3.  Отказоустойчивость message broker как single point of failure
    4.  Масштабирование BattleExecutorService по мере роста количества проводимых боев

Компоненты, к которым чаще всего будут меняться требования:

    1.  BattleExecutorService - по мере добавления новых механик и правил проведения боев
    2.  InfoService, UserHost, UserClients - по мере появления новой доступной пользователям информации и необходимости ее предоставления
